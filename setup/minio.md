# MinIO Setup Guide

MinIO serves as an S3-compatible Object Storage layer (our Data Lake), containing raw parquet logs generated by Spark.

## 1. Installation

Download the binary and set up the data directory:
```bash
cd ~
wget https://dl.min.io/server/minio/release/linux-amd64/minio
chmod +x minio
mkdir -p ~/MinIO/minio_data
```

## 2. Launching the MinIO Server

To avoid running long commands, use the helper script:

```bash
cd ~/eventsim_project/scripts
./run_minio.sh
```

### Manual Config Reference
The script maps these environments internally:
```bash
export MINIO_ROOT_USER=minioadmin
export MINIO_ROOT_PASSWORD=minioadmin
```
_Note: We use ports 9050 (API) and 9051 (Console) to avoid port-conflicts with Clickhouse on standard port 9000._

## 3. Creating the Bucket
1. Go to http://localhost:9051 in your browser.
2. Login with `minioadmin` / `minioadmin`.
3. Create a bucket named **`music_events`**.
4. Data should start appearing here after initializing Spark.

## 4. Notic
1. Delete all data in the bucket before running the pipeline to avoid error caused by checkpoint mechanism of spark
2. Delete ~/eventsim_project/checkpoint to avoid error caused by checkpoint of spark
