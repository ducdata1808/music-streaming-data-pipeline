Step 1: eventsim
- Clone repo: https://github.com/Interana/eventsim.git
- download java 8 and sbt:
    - echo "deb https://repo.scala-sbt.org/scalasbt/debian all main" | sudo tee /etc/apt/sources.list.d/sbt.list
    - echo "deb https://repo.scala-sbt.org/scalasbt/debian /" | sudo tee -a /etc/apt/sources.list.d/sbt.list
    - curl -sL "https://keyserver.ubuntu.com/pks/lookup?op=get&search=0x2EE0EA64E40A89B84B2DF73499E82A75642AC823" | sudo apt-key add
    - sudo apt-get update
    - sudo apt-get install openjdk-8-jdk scala sbt
- change java version:
    - sudo update-alternatives --config java
- run: sbt assembly
- grant permission for bin: chmod +x bin/eventsim
- check path: target/scala-2.12/eventsim-assembly-2.0.jar and change path in bin: nano bin/eventsim: #! /bin/bash 
java -XX:+AggressiveOpts -XX:+UseG1GC -XX:+UseStringDeduplication -Xmx4G -Duser.country=US -Duser.language=en -jar target/scala-2.12/eventsim-assembly-2.0.jar "$@"
- check eventsim by getting .json file: bin/eventsim -c "examples/example-config.json" -n 100 test_data.json

Step 6: ClickHouse
- curl https://clickhouse.com/ | sh
- sudo ./clickhouse install
- ~/clickhouse/clickhouse server
- turn on another terminal and run: clickhouse-client
- create database: CREATE DATABASE music_analytics;
- USE music_analytics;
- Xoá bảng nếu tồn tại: DROP TABLE IF EXISTS events;
CREATE TABLE events
(
    ts String,               
    userId String,
    sessionId UInt32,
    page String,
    auth String,
    method String,
    status UInt16,
    level String,
    itemInSession UInt32,
    location String,
    userAgent String,
    lastName String,
    firstName String,
    registration String,     
    gender String,
    artist String,
    song String,
    length Float32
)
ENGINE = S3('http://127.0.0.1:9050/music-events/data/*.parquet', 'minioadmin', 'minioadmin', 'Parquet');
- check: SELECT * FROM s3('http://127.0.0.1:9050/music-events/*.parquet', 'minioadmin', 'minioadmin', 'Parquet') 
- check number of columns: SELECT count() FROM events;
- SELECT 
    ts, 
    artist, 
    song, 
    firstName 
FROM events 
LIMIT 5;
- Problem: Spark write column ts as type INT96, but ClickHouse expect DateTime
- Solution: Convert ts to string in Spark and handle Datatime by DBT

Step 7: DBT
- download: activate venv and pip install -r requirements.txt
- check dbt: dbt --version
- create dbt project: dbt init music_events_transform
- edit profiles.yml: nano profiles.yml
    music_events_transform:
  outputs:
    dev:
      type: clickhouse
      host: localhost
      port: 8123
      user: default
      password: ""
      database: music_analytics
      secure: False
  target: dev
- cd music_events_transform
- dbt debug --profiles-dir ~/eventsim_project/music_events_transform : use profiles.yml in project folder instead of ~/.dbt/profiles.yml
- delete example folder: rm -r models/example
- create staging folder: mkdir models/staging -> clean raw data from clickhouse
- create marts folder: mkdir models/marts -> create final data for analysis
- create staging model: nano models/staging/stg_events.sql
- create sources.yml: nano models/staging/sources.yml
- cd music_events_transform
- dbt compile --profiles-dir ~/eventsim_project/music_events_transform
- dbt run --profiles-dir ~/eventsim_project/music_events_transform
- dbt test --profiles-dir ~/eventsim_project/music_events_transform
- after run sucessfully, check data in clickhouse: 
    SHOW TABLES IN music_analytics;
    SELECT * FROM stg_events LIMIT 5;

Step 8: Airflow
- download: pip install requirements.txt\
- copy DAG file to airflow DAGs folder: cp ~/eventsim_project/dag.py ~/airflow/dags/music_pipeline.py
- start airflow: airflow standalone
- create connnection:
        # MinIO connection
    airflow connections add 'minio_default' \
        --conn-type 'aws' \
        --conn-extra '{"endpoint_url": "http://localhost:9050", "aws_access_key_id": "minioadmin", "aws_secret_access_key": "minioadmin"}'
    # ClickHouse connection
    airflow connections add 'clickhouse_default' \
        --conn-type 'generic' \
        --conn-host 'localhost' \
        --conn-port 8123 \
        --conn-login 'default' \
        --conn-password '' \
        --conn-schema 'music_analytics'
    # Spark connection
    airflow connections add 'spark_default' \
        --conn-type 'spark' \
        --conn-host 'local' \
        --conn-extra '{"master": "local[*]"}'
- check connection: 
    airflow connections list | grep -E 'minio|clickhouse|spark'
- check services: jps
- check Airflow has received dag: 
    airflow dags list | grep music_pipeline
- check error: airflow dags list-import-errors
- trigger dag: airflow dags trigger music_events_pipeline
- view recent tasks: airflow dags list-runs -d music_events_pipeline
- check each task: airflow tasks states-for-dag-run music_events_pipeline <run_id>
